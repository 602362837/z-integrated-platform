# 一体化平台用户管理系统 - 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
建设一体化平台用户管理系统，统一管理平台用户和应用用户，实现组织机构统一管理及应用管理。

### 1.2 系统边界
- **本次建设**：用户管理、组织机构管理、应用管理
- **不在本次建设**：网关认证（由外部网关负责）

---

## 2. 用户分类模型

### 2.1 平台用户
- 统一平台的用户账号
- 平台唯一标识：`code`（全局唯一）
- 可登录平台管理后台

### 2.2 应用用户
- 各应用自己的用户
- 数据从子应用同步过来
- 每个应用用户必须映射到一个平台用户
- **平台用户code生成规则**：`{应用code}_{应用用户code}`

---

## 3. 功能模块

### 3.1 应用管理
| 功能 | 说明 |
|------|------|
| 应用列表 | 查看所有应用 |
| 新增应用 | 填写应用名称、应用code、描述 |
| 编辑应用 | 修改应用信息 |
| 删除应用 | 逻辑删除 |

**应用字段**：
- `id` - 主键
- `appName` - 应用名称
- `appCode` - 应用代码（唯一，用于拼接用户code）
- `description` - 描述
- `status` - 状态（启用/禁用）
- `createTime` - 创建时间

### 3.2 组织机构管理
| 功能 | 说明 |
|------|------|
| 组织列表 | 树形展示组织架构 |
| 新增组织 | 选择父级，填写组织名称、编码 |
| 编辑组织 | 修改组织信息 |
| 删除组织 | 删除时检查是否有子组织或用户 |
| 组织详情 | 查看组织下的用户 |

**组织字段**：
- `id` - 主键
- `orgName` - 组织名称
- `orgCode` - 组织编码（唯一）
- `parentId` - 父级ID（根节点为0）
- `sortOrder` - 排序
- `status` - 状态
- `createTime` - 创建时间

### 3.3 平台用户管理
| 功能 | 说明 |
|------|------|
| 用户列表 | 分页列表，支持搜索 |
| 新增用户 | 填写用户信息，关联组织 |
| 编辑用户 | 修改用户信息 |
| 删除用户 | 逻辑删除 |
| 用户详情 | 查看用户详细信息 |
| 分配角色 | 为用户分配角色（预留） |

**平台用户字段**：
- `id` - 主键
- `code` - 用户编码（全局唯一，系统生成）
- `username` - 用户名
- `nickname` - 昵称
- `password` - 密码（BCrypt加密存储）
- `email` - 邮箱
- `phone` - 手机号
- `orgId` - 所属组织ID
- `status` - 状态（启用/禁用）
- `createTime` - 创建时间

### 3.4 应用用户管理
| 功能 | 说明 |
|------|------|
| 应用用户列表 | 按应用筛选，显示映射的平台用户 |
| 同步用户 | 从应用同步用户数据 |
| 映射管理 | 手动建立/解除应用用户与平台用户的映射 |
| 用户详情 | 查看应用用户及对应平台用户信息 |

**应用用户字段**：
- `id` - 主键
- `appCode` - 应用代码
- `userCode` - 应用内用户编码
- `username` - 用户名
- `nickname` - 昵称
- `email` - 邮箱
- `phone` - 手机号
- `platformUserId` - 映射的平台用户ID
- `syncTime` - 最近同步时间
- `createTime` - 创建时间

---

## 4. 业务流程

### 4.1 应用用户同步流程
```
子应用 -> 同步接口 -> 应用用户表 -> 映射平台用户
                                    ↓
                            检查平台用户是否存在
                                    ↓
                    存在 -> 关联映射
                    不存在 -> 创建平台用户(code=appCode_userCode) -> 关联映射
```

### 4.2 用户code生成规则
- 平台用户：`{appCode}_{userCode}`（仅应用用户）
- 纯平台用户：`PLAT_{UUID}` 或用户自定义

## 5. 数据模型关系

```
┌─────────────┐     ┌─────────────┐
│   应用      │     │   组织      │
│  (App)      │     │  (Org)      │
└──────┬──────┘     └──────┬──────┘
       │                   │
       │                   │
       ▼                   ▼
┌─────────────────────────────────────┐
│           平台用户 (PlatformUser)    │
│  code: 唯一标识                     │
│  orgId: 所属组织                   │
└─────────────────────────────────────┘
              ▲
              │ 映射
              │
┌─────────────────────────────────────┐
│        应用用户 (AppUser)            │
│  appCode + userCode 联合唯一        │
│  platformUserId -> PlatformUser     │
└─────────────────────────────────────┘
```

## 6. API接口设计

### 6.1 应用管理
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/apps | 应用列表 |
| POST | /api/v1/apps | 新增应用 |
| PUT | /api/v1/apps/{id} | 编辑应用 |
| DELETE | /api/v1/apps/{id} | 删除应用 |

### 6.2 组织管理
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/orgs | 组织树形列表 |
| POST | /api/v1/orgs | 新增组织 |
| PUT | /api/v1/orgs/{id} | 编辑组织 |
| DELETE | /api/v1/orgs/{id} | 删除组织 |

### 6.3 平台用户管理
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/platform-users | 用户列表 |
| POST | /api/v1/platform-users | 新增用户 |
| PUT | /api/v1/platform-users/{id} | 编辑用户 |
| DELETE | /api/v1/platform-users/{id} | 删除用户 |
| GET | /api/v1/platform-users/{id} | 用户详情 |

### 6.4 应用用户管理
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/app-users | 应用用户列表 |
| POST | /api/v1/app-users/sync | 同步用户 |
| POST | /api/v1/app-users/{id}/map | 映射到平台用户 |
| DELETE | /api/v1/app-users/{id}/map | 解除映射 |
