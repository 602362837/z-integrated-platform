# 数据字典

## 概述

本文档详细说明一体化平台用户管理系统数据库表结构。

---

## t_app 应用表

### 表结构说明

存储多应用的基本信息，每个应用对应一个独立的业务系统。

| 字段名 | 数据类型 | 是否为空 | 默认值 | 说明 |
|--------|----------|----------|--------|------|
| id | BIGINT | NOT NULL | 自增 | 主键 |
| app_name | VARCHAR(100) | NOT NULL | - | 应用名称 |
| app_code | VARCHAR(50) | NOT NULL | - | 应用代码（唯一） |
| description | VARCHAR(500) | NULL | NULL | 描述 |
| status | TINYINT | NOT NULL | 1 | 状态：0-禁用，1-启用 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| deleted | TINYINT | NOT NULL | 0 | 逻辑删除：0-未删除，1-已删除 |

### 字段详细说明

#### id
- 类型：BIGINT (19位整数)
- 说明：主键，自增序列
- 约束：主键 (PRIMARY KEY)

#### app_name
- 类型：VARCHAR(100)
- 说明：应用名称，用于展示
- 约束：NOT NULL
- 示例：订单管理系统

#### app_code
- 类型：VARCHAR(50)
- 说明：应用代码，系统内部唯一标识
- 约束：NOT NULL，唯一键 (UNIQUE KEY uk_app_code)
- 示例：oms, wms, crm

#### description
- 类型：VARCHAR(500)
- 说明：应用描述信息
- 约束：可为空
- 示例：订单管理应用

#### status
- 类型：TINYINT (1位整数)
- 说明：应用状态
- 约束：NOT NULL，默认值 1
- 取值：
  - 0：禁用
  - 1：启用

#### create_time
- 类型：DATETIME
- 说明：记录创建时间
- 约束：NOT NULL，默认值 CURRENT_TIMESTAMP

#### update_time
- 类型：DATETIME
- 说明：记录最后更新时间
- 约束：NOT NULL，自动更新

#### deleted
- 类型：TINYINT (1位整数)
- 说明：逻辑删除标志
- 约束：NOT NULL，默认值 0
- 取值：
  - 0：未删除
  - 1：已删除

## t_org 组织表

### 表结构说明

存储组织机构信息，支持多级树形结构。

| 字段名 | 数据类型 | 是否为空 | 默认值 | 说明 |
|--------|----------|----------|--------|------|
| id | BIGINT | NOT NULL | 自增 | 主键 |
| org_name | VARCHAR(100) | NOT NULL | - | 组织名称 |
| org_code | VARCHAR(50) | NOT NULL | - | 组织编码（唯一） |
| parent_id | BIGINT | NOT NULL | 0 | 父级ID：0-根节点 |
| sort_order | INT | NOT NULL | 0 | 排序号 |
| status | TINYINT | NOT NULL | 1 | 状态：0-禁用，1-启用 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| deleted | TINYINT | NOT NULL | 0 | 逻辑删除：0-未删除，1-已删除 |

### 字段详细说明

#### id
- 类型：BIGINT (19位整数)
- 说明：主键，自增序列
- 约束：主键 (PRIMARY KEY)

#### org_name
- 类型：VARCHAR(100)
- 说明：组织名称，用于展示
- 约束：NOT NULL
- 示例：研发中心、产品部门

#### org_code
- 类型：VARCHAR(50)
- 说明：组织编码，系统内部唯一标识
- 约束：NOT NULL，唯一键 (UNIQUE KEY uk_org_code)
- 示例：HQ, RD, PM, QA

#### parent_id
- 类型：BIGINT (19位整数)
- 说明：父级组织ID，用于构建树形结构
- 约束：NOT NULL，默认值 0 表示根节点
- 约束：索引 (KEY idx_parent_id)
- 示例：1 表示隶属于ID=1的组织

#### sort_order
- 类型：INT (10位整数)
- 说明：排序号，用于列表展示顺序
- 约束：NOT NULL，默认值 0
- 示例：1, 2, 3

#### status
- 类型：TINYINT (1位整数)
- 说明：组织状态
- 约束：NOT NULL，默认值 1
- 取值：
  - 0：禁用
  - 1：启用

#### create_time
- 类型：DATETIME
- 说明：记录创建时间
- 约束：NOT NULL，默认值 CURRENT_TIMESTAMP

#### update_time
- 类型：DATETIME
- 说明：记录最后更新时间
- 约束：NOT NULL，自动更新

#### deleted
- 类型：TINYINT (1位整数)
- 说明：逻辑删除标志
- 约束：NOT NULL，默认值 0
- 取值：
  - 0：未删除
  - 1：已删除

## t_platform_user 平台用户表

### 表结构说明

存储平台级用户信息，是系统的超级用户，拥有管理所有应用的权限。

| 字段名 | 数据类型 | 是否为空 | 默认值 | 说明 |
|--------|----------|----------|--------|------|
| id | BIGINT | NOT NULL | 自增 | 主键 |
| code | VARCHAR(100) | NOT NULL | - | 用户编码（全局唯一） |
| username | VARCHAR(50) | NOT NULL | - | 用户名 |
| nickname | VARCHAR(50) | NULL | NULL | 昵称 |
| password | VARCHAR(255) | NOT NULL | - | 密码（BCrypt加密） |
| email | VARCHAR(100) | NULL | NULL | 邮箱 |
| phone | VARCHAR(20) | NULL | NULL | 手机号 |
| org_id | BIGINT | NULL | NULL | 所属组织ID |
| status | TINYINT | NOT NULL | 1 | 状态：0-禁用，1-启用 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| deleted | TINYINT | NOT NULL | 0 | 逻辑删除：0-未删除，1-已删除 |

### 字段详细说明

#### id
- 类型：BIGINT (19位整数)
- 说明：主键，自增序列
- 约束：主键 (PRIMARY KEY)

#### code
- 类型：VARCHAR(100)
- 说明：用户编码，全局唯一标识
- 约束：NOT NULL，唯一键 (UNIQUE KEY uk_code)
- 示例：PLAT_ADMIN001

#### username
- 类型：VARCHAR(50)
- 说明：用户名，用于登录
- 约束：NOT NULL，唯一键 (UNIQUE KEY uk_username)
- 示例：admin

#### nickname
- 类型：VARCHAR(50)
- 说明：用户昵称，用于展示
- 约束：可为空
- 示例：系统管理员

#### password
- 类型：VARCHAR(255)
- 说明：登录密码，使用BCrypt加密存储
- 约束：NOT NULL
- 注意：密码加密算法为BCrypt，长度60字符

#### email
- 类型：VARCHAR(100)
- 说明：用户邮箱
- 约束：可为空
- 示例：admin@platform.com

#### phone
- 类型：VARCHAR(20)
- 说明：用户手机号
- 约束：可为空
- 示例：13800138000

#### org_id
- 类型：BIGINT (19位整数)
- 说明：所属组织ID，关联t_org表
- 约束：可为空，索引 (KEY idx_org_id)
- 示例：1

#### status
- 类型：TINYINT (1位整数)
- 说明：用户状态
- 约束：NOT NULL，默认值 1
- 取值：
  - 0：禁用
  - 1：启用

#### create_time
- 类型：DATETIME
- 说明：记录创建时间
- 约束：NOT NULL，默认值 CURRENT_TIMESTAMP

#### update_time
- 类型：DATETIME
- 说明：记录最后更新时间
- 约束：NOT NULL，自动更新

#### deleted
- 类型：TINYINT (1位整数)
- 说明：逻辑删除标志
- 约束：NOT NULL，默认值 0
- 取值：
  - 0：未删除
  - 1：已删除

## t_app_user 应用用户表

### 表结构说明

存储各个应用的用户信息，通过platform_user_id关联平台用户，实现用户统一管理。

| 字段名 | 数据类型 | 是否为空 | 默认值 | 说明 |
|--------|----------|----------|--------|------|
| id | BIGINT | NOT NULL | 自增 | 主键 |
| app_code | VARCHAR(50) | NOT NULL | - | 应用代码 |
| user_code | VARCHAR(100) | NOT NULL | - | 应用内用户编码 |
| username | VARCHAR(50) | NULL | NULL | 用户名 |
| nickname | VARCHAR(50) | NULL | NULL | 昵称 |
| email | VARCHAR(100) | NULL | NULL | 邮箱 |
| phone | VARCHAR(20) | NULL | NULL | 手机号 |
| platform_user_id | BIGINT | NULL | NULL | 映射的平台用户ID |
| sync_time | DATETIME | NULL | NULL | 最近同步时间 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| deleted | TINYINT | NOT NULL | 0 | 逻辑删除：0-未删除，1-已删除 |

### 字段详细说明

#### id
- 类型：BIGINT (19位整数)
- 说明：主键，自增序列
- 约束：主键 (PRIMARY KEY)

#### app_code
- 类型：VARCHAR(50)
- 说明：应用代码，关联t_app表
- 约束：NOT NULL
- 示例：oms, wms, crm

#### user_code
- 类型：VARCHAR(100)
- 说明：应用内用户编码，在各自应用内唯一
- 约束：NOT NULL
- 示例：USER_001

#### username
- 类型：VARCHAR(50)
- 说明：应用内用户名
- 约束：可为空
- 注意：与应用内实际用户名保持一致

#### nickname
- 类型：VARCHAR(50)
- 说明：应用内用户昵称
- 约束：可为空
- 示例：张三

#### email
- 类型：VARCHAR(100)
- 说明：用户邮箱
- 约束：可为空

#### phone
- 类型：VARCHAR(20)
- 说明：用户手机号
- 约束：可为空

#### platform_user_id
- 类型：BIGINT (19位整数)
- 说明：关联的平台用户ID，实现用户映射
- 约束：可为空，索引 (KEY idx_platform_user_id)
- 说明：非空时表示该应用用户关联了平台用户

#### sync_time
- 类型：DATETIME
- 说明：最近一次同步时间
- 约束：可为空
- 说明：当平台用户信息变更时，同步更新应用用户

#### create_time
- 类型：DATETIME
- 说明：记录创建时间
- 约束：NOT NULL，默认值 CURRENT_TIMESTAMP

#### update_time
- 类型：DATETIME
- 说明：记录最后更新时间
- 约束：NOT NULL，自动更新

#### deleted
- 类型：TINYINT (1位整数)
- 说明：逻辑删除标志
- 约束：NOT NULL，默认值 0
- 取值：
  - 0：未删除
  - 1：已删除

## 表关系说明

```
t_app (应用表)
    │
    └─ t_app_user (应用用户表)
              │
              └─ t_platform_user (平台用户表)
                              │
                              └─ t_org (组织表)
```

### 关系描述

1. **t_platform_user -> t_org**: 多对一关系，一个用户属于一个组织
2. **t_app_user -> t_platform_user**: 多对一关系，一个应用用户可映射到一个平台用户
3. **t_app_user -> t_app**: 多对一关系，一个应用用户属于一个应用
4. **t_org 自关联**: 一对多关系，组织支持树形结构
